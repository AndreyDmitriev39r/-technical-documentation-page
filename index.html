Algorithmic complexity with examples

General information

Before we jump into specific examples, let's refresh some general knowledge.

  • Algorithm is just a set of steps (or one could say an order of actions) that need to be taken to finish some task. You can think of it as of instruction or recipe also.
  • Efficiency of an algorithm can be evaluated by measuring resources which this algorithm uses to solve particular task.
  • Asymptotic notation (also called Big O notation) is a system that allows us to analyze and compare the performance of an algorithm as it's input grows.
  • Don't get confused by effectiveness and efficiency. Effective algorithm - one that solves the problem. Efficient algoritm - one that does the same spending small amount of resources.
  • When evaluating efficiency, we can try to measure time of execution for our program, but in this case results will depend on configuration of our hardware.

And now, with all that said, let's move on!

Constant complexity O(1)

When one says that algorithm has constant complexity, it means that number of operations won't change, no matter how big input is. Let's say we need to find sum of all natural numbers from 1 to n. There is a nice little formula for that. 

          
            const sumOfNaturals = (n) => {
              return n * (n + 1) / 2;
            }

In the above example it doesn't matter, how big n is. Number of operations will be the same.
